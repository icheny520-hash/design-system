设计原则
✦
一致性
所有组件共享同一套视觉语言，颜色、圆角、间距均来自 Token，确保跨场景一致。
◈
可访问性
满足 WCAG 2.1 AA 标准，对比度不低于 4.5:1，支持键盘导航与屏幕阅读器。
⬡
模块化
每个组件独立封装，可自由组合，通过 Props 扩展变体，不依赖外部全局状态。
◎
响应式

属性 API
属性类型默认值必填说明
variantstring"primary"可选视觉变体：primary / secondary / ghost / danger / outline / link
sizestring"md"可选尺寸：sm / md / lg / xl
iconReactNode—可选图标节点，默认置于文字左侧
iconPositionstring"left"可选图标位置：left / right
iconOnlybooleanfalse可选纯图标模式，需配合 aria-label 提供无障碍文本
disabledbooleanfalse可选禁用状态，阻止所有交互

menuMenuItem[]—可选Button.Dropdown / Split 的菜单项配置
onClickfunction—可选点击回调，接收原生事件对象
交互行为
图标对齐
图标与文字垂直居中对齐，间距固定 6px。纯图标按钮宽高相等，呈正方形。
下拉定位
下拉菜单默认左对齐触发按钮，超出视口右侧时自动翻转为右对齐；点击菜单外区域关闭。
分裂按钮
左侧执行默认操作，右侧箭头区域独立触发下拉，两个区域的焦点状态和点击区域完全分离。
按钮组
相邻按钮共享边框，避免双边框；激活项使用主色背景，非激活项保持次要样式。
